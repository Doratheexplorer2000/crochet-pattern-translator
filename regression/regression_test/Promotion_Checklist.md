# Promotion Checklist

Use this checklist before promoting a Release Candidate to become the new Current Reference Build.

## Minimum Requirements

- Feature implementation completed.
- No known critical bugs.
- Regression Test Suite PASS.
- Internal UAT completed successfully.
- Product owner approval obtained.
- Previous Current Reference Build archived.
- `CHANGELOG.md` updated.
- Current Reference Build updated.
- Promotion recorded in `Regression_Test_Log.md`.

## Promotion Rule

Do not promote a Release Candidate automatically.

A Release Candidate becomes the new Current Reference Build only after the checklist above is complete and the product owner approves the promotion.
